# 08 — Policies and Guards

Forge must be safe enough to improve itself.

## Policy principle

Agents may propose. Forge verifies.

## Write policy

Allowed paths in early bootstrap:

```text
src/
tests/
skills/
.forge/state/
.forge/memory/
.forge/backlog/
README.md
```

Protected paths:

```text
.git/
.env
.github/
.devcontainer/
AGENTS.md
forge.yml
.forge/horizon/
.forge/policy/
```

## Path traversal guard

Reject paths containing:

```text
../
..\
absolute paths
symlink escapes
```

## Shell command policy

Allowed:

```text
static commands from forge.yml
known internal Forge commands
```

Rejected:

```text
commands generated by model output
commands from issue text
commands that expose environment variables
network commands unless explicitly enabled
```

## Git policy

Initial version:

```text
do not commit automatically unless explicitly requested
do not push automatically
do not merge automatically
```

Self-improvement version:

```text
create branch
make change
run tests
create commit
human reviews merge
```

## Risk levels

### Low risk

```text
docs
tests
status output
run artifact formatting
memory updates
```

### Medium risk

```text
orchestration logic
context selection
Codex runner
CLI behavior
```

### High risk

```text
policy changes
protected file changes
GitHub Actions
secret handling
auto-merge
dependency upgrades
sandbox changes
```

## High-risk rule

High-risk tasks require explicit human approval and must not be selected automatically by `forge improve`.

## Verification gate

A run is not successful unless:

```text
required artifacts exist
tests pass
protected files unchanged
lessons or outcome written
state is not contradictory
```
